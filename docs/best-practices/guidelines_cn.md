# 提示词工程最佳实践\n\n[English](guidelines.md) | 中文\n\n## 通用指导原则\n\n### 1. 具体明确\n\n**❌ 避免：**\n```\n\"帮我做演示文稿\"\n```\n\n**✅ 更好：**\n```\n\"帮我为企业受众创建一个关于可再生能源好处的15分钟演示文稿大纲，包括3个主要观点和支持数据建议。\"\n```\n\n### 2. 提供充分的上下文\n\n**❌ 避免：**\n```\n\"修复这段代码\"\n```\n\n**✅ 更好：**\n```\n\"修复这个应该计算复利但返回错误值的Python函数。函数应该接受本金、利率和时间作为参数：\n\n[代码在这里]\n\n期望值：本金=$1000，利率=5%，时间=10年时为$1,628.89\n实际值：$1,500.00\"\n```\n\n### 3. 在有帮助时使用示例\n\n**少样本提示示例：**\n```\n\"将这些产品名称转换为URL友好的标识符：\n\n产品：'优质咖啡豆（深度烘焙）'\n标识符：premium-coffee-beans-dark-roast\n\n产品：'无线蓝牙耳机 - 降噪'\n标识符：wireless-bluetooth-headphones-noise-cancelling\n\n产品：'有机绿茶蜂蜜混合'\n标识符：\"\n```\n\n### 4. 结构化您的提示词\n\n对于复杂的提示词使用清晰的部分：\n\n```\n**角色：** 您是一位经验丰富的UX设计师\n\n**任务：** 审查这个移动应用线框图并提供反馈\n\n**上下文：** 这是一个针对忙碌专业人士的食品配送应用\n\n**重点领域：**\n- 导航清晰度\n- 用户流程效率\n- 无障碍考虑\n\n**输出格式：** \n- 问题列表（如有）\n- 具体改进建议\n- 总体评分（1-10）\n```\n\n## 特定领域最佳实践\n\n### 写作和内容创作\n\n1. **指定语调和风格**\n   ```\n   \"以适合博客受众的对话式、友好语调写作\"\n   ```\n\n2. **定义目标受众**\n   ```\n   \"目标受众：技术知识有限的小企业主\"\n   ```\n\n3. **设置长度约束**\n   ```\n   \"写一篇包含3个主要部分的300字文章\"\n   ```\n\n4. **包含SEO要求**\n   ```\n   \"自然地包含关键词'可持续包装'3-4次\"\n   ```\n\n### 代码和技术任务\n\n1. **指定编程语言和版本**\n   ```\n   \"编写一个使用类型提示的Python 3.9+函数\"\n   ```\n\n2. **包含要求和约束**\n   ```\n   \"函数应该处理边缘情况并包含错误处理\"\n   ```\n\n3. **提供期望的输入/输出**\n   ```\n   \"输入：整数列表\n   输出：频率计数字典\n   示例：[1,2,2,3] → {1:1, 2:2, 3:1}\"\n   ```\n\n4. **提及编码标准**\n   ```\n   \"遵循PEP 8风格指南并包含文档字符串\"\n   ```\n\n### 数据分析\n\n1. **描述数据结构**\n   ```\n   \"数据集包含1000行，列为：日期、销售额、地区、产品\"\n   ```\n\n2. **指定分析目标**\n   ```\n   \"识别季节性趋势和表现最佳的地区\"\n   ```\n\n3. **请求特定输出**\n   ```\n   \"提供汇总统计、可视化建议和关键洞察\"\n   ```\n\n### 商业和专业\n\n1. **定义商业背景**\n   ```\n   \"针对50-200员工目标市场的B2B SaaS公司\"\n   ```\n\n2. **指定利益相关者受众**\n   ```\n   \"此报告将向C级高管展示\"\n   ```\n\n3. **包含行业考虑**\n   ```\n   \"考虑医疗保健行业法规和合规要求\"\n   ```\n\n## 高级技术\n\n### 1. 提示词链\n\n将复杂任务分解为顺序步骤：\n\n```\n步骤1：\"分析这些客户反馈数据并识别前5个投诉类别\"\n\n步骤2：\"对于每个识别的投诉类别，建议具体的改进行动\"\n\n步骤3：\"根据影响和实施难度对这些改进进行优先排序\"\n```\n\n### 2. 条件逻辑\n\n使用if-then语句进行动态响应：\n\n```\n\"审查这段代码并：\n- 如果有语法错误，列出它们及行号\n- 如果代码语法正确但有逻辑问题，解释问题\n- 如果代码正确，建议优化\n- 如果代码已经最优，确认这一点并解释原因\"\n```\n\n### 3. 多角度分析\n\n```\n\"从三个角度分析这个商业提案：\n1. 财务分析师：关注ROI和成本效益\n2. 风险管理者：识别潜在风险和缓解策略\n3. 运营经理：评估实施可行性\n\n基于所有三个角度提供总结建议。\"\n```\n\n### 4. 迭代改进\n\n```\n\"为我们的新产品创建营销邮件。在您提供第一稿后，我会给您反馈，然后您可以创建改进版本。\"\n```\n\n## 质量保证\n\n### 测试您的提示词\n\n1. **用边缘情况测试**\n   - 空输入\n   - 异常格式\n   - 极值\n\n2. **验证一致性**\n   - 多次运行相同提示词\n   - 检查质量一致性\n\n3. **跨不同模型测试**\n   - 不同AI模型可能响应不同\n   - 调整提示词以获得最佳性能\n\n### 常见问题和解决方案\n\n| 问题 | 解决方案 |\n|------|----------|\n| 模糊响应 | 添加更具体的指令 |\n| 错误格式 | 明确指定输出格式 |\n| 不完整答案 | 分解为更小、更集中的提示词 |\n| 质量不一致 | 添加示例和约束 |\n| 偏离主题的响应 | 提供更清晰的上下文和边界 |\n\n## 伦理考虑\n\n### 1. 避免有害内容\n- 不要请求可能造成伤害的内容\n- 注意提示词中的偏见\n- 考虑生成内容的影响\n\n### 2. 尊重隐私\n- 不要包含个人或敏感信息\n- 谨慎处理专有数据\n- 遵循数据保护指南\n\n### 3. 透明度\n- 明确说明AI生成的内容\n- 不要误传AI能力\n- 承认局限性\n\n## 性能优化\n\n### 1. 令牌效率\n- 删除不必要的词语\n- 使用清晰、简洁的语言\n- 避免重复\n\n### 2. 响应时间\n- 较短的提示词通常处理更快\n- 将复杂任务分解为较小部分\n- 为任务使用适当的模型\n\n### 3. 成本管理\n- 监控令牌使用\n- 优化提示词长度\n- 对重复查询使用缓存\n\n## 持续改进\n\n### 1. 记录有效方法\n- 保持有效提示词库\n- 注意哪些变化表现更好\n- 跟踪成功指标\n\n### 2. 保持更新\n- 关注AI模型更新\n- 从社区最佳实践中学习\n- 尝试新技术\n\n### 3. 收集反馈\n- 获取用户对输出的反馈\n- 根据业务目标衡量\n- 基于结果进行迭代\n\n## 快速参考检查清单\n\n**提交提示词前，检查：**\n\n- [ ] 指令是否清晰具体？\n- [ ] 我是否提供了充分的上下文？\n- [ ] 是否指定了期望的输出格式？\n- [ ] 是否提到了任何约束或要求？\n- [ ] 示例是否有助于澄清任务？\n- [ ] 提示词范围是否适当（不太宽泛/狭窄）？\n- [ ] 我是否考虑了潜在的边缘情况？\n- [ ] 语言是否专业和适当？\n- [ ] 这个提示词是否可能产生期望的结果？\n\n## 下一步\n\n- 练习[示例](../../examples/)\n- 使用[模板](../../templates/)作为起点\n- 探索[高级技术](../techniques/)\n- 尝试[开发工具](../../tools/)\n"
